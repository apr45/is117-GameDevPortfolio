# AGENTS.md — Project Directive & AI Instructions

### 1. The Mission
You are assisting a developer and IT student at NJIT in building a professional, aspirational scrollytelling portfolio. The goal is to represent the user as an emerging **Video Game Developer/Designer**.

### 2. Core Identity & Branding
* **Brand Archetypes:** * **The Creator:** Focus on the act of building, technical mastery, and structural innovation.
    * **The Magician:** Focus on transformation, "magic" moments in gameplay/code, and captivating transitions.
* **Design Aesthetic:** 8-bit / Retro Video Game aesthetic. Simple, clean, and visual-heavy.
* **Psychological Framework:** * **Unity:** Creating a shared identity with the tech/gaming community.
    * **Liking:** Building a personal connection through relatable photos and "journey-based" storytelling.

### 3. Technical Constraints
* **Frameworks:** TypeScript, JSX (React/Vite).
* **Deployment:** GitHub Pages.
* **Architecture:** Static one-page "Scrollytelling."
* **Structural Elements:**
    * **Sticky Hero:** Remains pinned as content scrolls over it.
    * **Horizontal Scene:** A specific section that transitions from vertical to horizontal scroll (Project Showcase).
    * **Call to Action:** Sticky or footer-based contact point.

### 4. Content Guidelines
* **Word Count:** Maintain a 100–200 word limit per section. 
* **Tone:** Professional yet aspirational. It should tell the story of the "path to the dream."
* **Visuals:** Prioritize placeholders for high-quality photos of the developer and project screenshots.

### 6. Audience & Tone
* **Primary Audience:** * Fellow IT/CS students and members of the Game Dev club (peers).
    * Potential collaborators looking for specialized partners for game projects.
* **Tone:** Professional but engaging. It must grab attention within the first few seconds (Hero) and maintain engagement through interactive "scrollytelling" mechanics. 
* **Voice:** Insider/Developer-focused. Use industry-standard terminology (e.g., "Physics Implementation," "State Management") to build credibility.

### 5. Instruction for Future AI Interactions
When generating code or suggestions:
1.  **Prioritize TypeScript:** Ensure all code snippets use TS interfaces and types.
2.  **Scrollytelling Context:** Assume every component exists within a scroll-linked timeline.
3.  **8-Bit Styling:** Suggest CSS (Tailwind or Modules) that utilizes pixel-fonts, high-contrast borders, and "gaming" UI patterns.
4.  **No Bridge Phrases:** Do not justify your suggestions with "Based on your interest in..."—execute the vision directly and concisely.
